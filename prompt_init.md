# ЗАДАЧА / # GOAL
Разработать Telegram-бота для начальника цеха, который позволит устанавливать задания для сотрудников, отслеживать их выполнение и формировать отчеты о работе. Бот должен обеспечивать простой интерфейс для выбора оборудования, продукции и ответственных сотрудников, а также подтверждение выполнения заданий.

# ТРЕБОВАНИЯ / # REQUIREMENTS
1. Начальник цеха должен иметь возможность:
   - Выбирать оборудование из предустановленного списка.
   - Выбирать продукцию из предустановленного списка.
   - Устанавливать количество продукции.
   - Назначать ответственного сотрудника из списка.
   
2. Сотрудники должны иметь возможность:
   - Получать индивидуальные задания.
   - Подтверждать получение заданий.
   - Отчитываться о фактически выполненной продукции в конце смены.
   
3. Начальник должен получать уведомления о закрытии заданий сотрудниками и иметь возможность формировать отчет с информацией о статусе заданий, плановом и фактическом количестве.

# TECH STACK
- **Язык программирования**: Python — для разработки бота, благодаря его простоте и широкому сообществу.
- **Библиотека**: python-telegram-bot — для интеграции с Telegram API, простота использования и хорошая документация.
- **База данных**: SQLite — для хранения заданий и данных сотрудников, подходит для небольшого числа пользователей и простоты развертывания.
- **Веб-фреймворк**: Flask — для создания REST API, что обеспечит возможность расширения функционала в будущем.

# АРХИТЕКТУРА
1. **Микросервисная архитектура**:
   - Telegram-бот как основной сервис, обрабатывающий команды и взаимодействующий с пользователями.
   - REST API для взаимодействия с базой данных, обеспечивающий доступ к заданиям и пользователям.
   
2. **Паттерн 'Command'**:
   - Использование паттерна для обработки различных команд (например, создание задания, подтверждение, отчет) от начальника и сотрудников.

# ДОПОЛНИТЕЛЬНЫЕ ТРЕБОВАНИЯ
- Обеспечение соответствия требованиям ФЗ 152 РФ о персональных данных:
   - Шифрование данных.
   - Соблюдение правил хранения и обработки персональных данных.
- Логирование операций.
- Роли: Администратор, Начальник, Сотрудник.
- Оборудование имеет принадлежноть к оспределенному участку.
- Продукция может производиться на одном или нескольких оборудовании, но имеет оборудование по умолчанию.
- Задание устанавливается на одну смену (1-я смена: с 8:00 до 20:00, 2-я смена: с 20:00 до 8:00).
- Реализация механизма уведомлений для начальника о закрытии заданий.
- ТГ-бот будет развернут на VPS.
- Ключи должны размещаться в .env файле.

# OUTPUT FORMAT
1. Код проекта должен быть структурирован следующим образом:
   - `bot.py` — основной файл с логикой бота.
   - `api.py` — файл с реализацией REST API.
   - `models.py` — файл с моделями данных для базы данных.
   - `requirements.txt` — файл с зависимостями проекта.
   
2. Документация к API и инструкции по развертыванию должны быть включены в проект:
   - README.md — с описанием установки и использования.
   - Swagger или аналогичный инструмент для документирования API. 

3. Отчеты должны генерироваться в формате CSV или PDF, чтобы их можно было легко экспортировать и анализировать.
